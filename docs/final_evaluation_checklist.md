# Checklist d'evaluation finale - Flutter debutant

Cette grille permet de valider les acquis a la fin de la formation.

## Mode d'evaluation
- 0 = Non acquis
- 1 = Partiellement acquis
- 2 = Acquis

Score cible recommande: **au moins 22/30** avec aucun critere critique a 0.

## Grille de competences

### 1) Fondamentaux Flutter (critique)
- [ ] Demarre et explique la structure d'une application Flutter de base.
- [ ] Utilise correctement les widgets de layout (`Column`, `Row`, `Expanded`, `ListView`).
- [ ] Produit une interface lisible avec un theme coherent.

### 2) Authentification debutant (critique)
- [ ] Implemente inscription avec validation de formulaire.
- [ ] Implemente connexion avec verification des champs.
- [ ] Gere session locale et deconnexion propre.

### 3) Navigation et reutilisation
- [ ] Met en place une navigation login -> app -> logout.
- [ ] Protege l'acces a l'ecran principal si non connecte.
- [ ] Cree au moins un widget reutilisable proprement isole.

### 4) Etat local et formulaires (critique)
- [ ] Met a jour l'interface avec `setState` sans comportement incoherent.
- [ ] Implemente le formulaire d'ajout de depense avec validation simple.
- [ ] Affiche des messages d'erreur et de confirmation utiles.

### 5) Donnees et architecture (critique)
- [ ] Modelise correctement une entite `Expense`.
- [ ] Sauvegarde et recharge les donnees locales.
- [ ] Relie les depenses a l'utilisateur connecte.
- [ ] Organise le code en couches simples (`models`, `screens`, `widgets`, `services`).

### 6) Robustesse et qualite
- [ ] Gere au moins un etat vide et un etat erreur.
- [ ] Ecrit au minimum 3 tests widgets pertinents.
- [ ] Sait expliquer ses choix techniques principaux.

## Validation fonctionnelle du projet final
- [ ] Inscription utilisateur fonctionnelle.
- [ ] Connexion utilisateur fonctionnelle.
- [ ] Deconnexion utilisateur fonctionnelle.
- [ ] Ajout d'une depense fonctionnel.
- [ ] Suppression d'une depense fonctionnelle.
- [ ] Filtre simple operationnel.
- [ ] Total mensuel calcule correctement.
- [ ] Application stable lors d'une demo complete.

## Decision finale
- [ ] Formation validee.
- [ ] Formation a renforcer (preciser les points faibles).
